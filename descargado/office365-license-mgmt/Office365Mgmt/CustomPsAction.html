How to write Action query?

  • Every Action has two kind of inputs, 1. Action Objects (selected rows from report grid), 2. Action Target Values (any values using macros).
  • You need to put the following line at the top of action query which is used to provide required column to select from the selected rows in the final part of the macro. 
    $objectArray = [Macro_ActionObjects_UserPrincipalName] -> Used to read column value of the selected rows
  • You need to iterate $objectArray using ForEach. Inside ForEach, you need to write the required action query.
  • Finally, you need to write following query to get action results 
    New-Object -TypeName PSObject -Property @{
Name=$_
Status=if($error.count -eq 0) {'Success'} Else {'Failed'}
Message=if($error.count -eq 0){ 'Success' }else{ $error[0].ToString(); }}

    How to get user inputs at runtime in Custom Action?

  • You can give inputs for every Action on runtime by using Macros, you can use the below listed macros to get runtime user inputs.
  • General Macro format: The Macros are should start with '[Macro_' and Ends with ']'. The text which is in front of control is followed by macro start text, then control type like Int, TextBox, Browse,...etc. and then ENDTEXT or ET for the end text which is followed by the control.
  • TextBox: If you need text input, add below macro to load TextBox control. 
    [Macro_UserName_Text]                          -> Load TextBox followed by UserName.
  • SecureText: If you need text input which is never show, add below macro to load secure TextBox control, the typing characters are shows '*'. 
    [Macro_Password_SecureText]                          -> Load secure TextBox followed by Password.
  • Integer Control: If you need number as input, add below macro to load Numeric control. 
    [Macro_Age_Int]                      -> Include Numeric control followed by text 'Age'
    [Macro_Inactive in last_Int_et:days] -> Include Numeric control with End text 'days'
  • CheckBox Control: If you need boolean as input, add below macro to load Checkbox control. 
    [Macro_Enable_CheckBox]                      -> Include Checkbox control in front of text 'Enable'
    [Macro_Enable_CheckBox_DV:True]              -> Include Checkbox control with checked state
  • DropDown: This macro loads DropDown control. You need to provide dropdown items as semi-colon separated values with the parameter 'Options:'. 
    [Macro_IsLicensed_DropDown_Options:True;False] -> DropDown control loads 'True' and 'False' from this Macro.
  • DateTime Control: You can add Date, Time, Datetime and DateRange controls by using the following macros. 
    [Macro_DOB_DateTime]                 -> Include DateTime control followed by 'DOB' and return DateTime.
    [Macro_To_DateRange]                 -> if you need two dates, use this macro along with DateTime macro to get Start and End datetime. 
    [Macro_DOB_Date]                     -> this will return only date value.
    [Macro_DOB_Time]                     -> this will return only time value.
  • Target: This control get values from the given target and populate values in dropdown. You can select value from dropdown. 
    [Macro_User Name_Target_ADUserInfo_DisplayName]-> Dropdown items loads DisplayName from ADUserInfo objetcs from Database.
  • Object Picker: This control provides Browse option to select any target from database. If you need to provide Browse option to select multiple objects. The last part of the macro should include 'True' 
    [Macro_Name_Browse_ADUserInfo_DisplayName]    -> Provide option to Browse objects from ADUserInfo in Database.
    [Macro_Name_Browse_MailBoxInfo_DisplayName]   ->  Provide option to Browse objects from MailBoxInfo in Database.
  • User Picker: This control provides Browse option to select users from the target ADUserInfo in database. You need to provide required column to select in the final part of the macro. The last part of the macro 'True' decides whether this is single or multi valued. By default it is single value. 
    [Macro_User_UserPicker_UserPrincipalName]     -> Provide option to browse user from ADUserInfo and return single 'UserPrincipalName'
    [Macro_User_UserPicker_UserPrincipalName_True]-> Provide option to browse users from ADUserInfo and return 'UserPrincipalName' collection as Powershell Array.
  • AD-Object Picker: This control provides Browse option to select any objects from active directory. If you need to provide Browse option to select multiple objects. The last part of the macro should include 'True' 
    [Macro_Name_ADObjectPicker_User_Name]    -> Provide option to Browse user objects from AD.
    [Macro_Name_ADObjectPicker_Group_ObjectGuid_true]   ->  Provide option to Browse group objects guid from AD.
  • OU Picker: This control provides Browse option to select ou's from active directory. You need to provide required column to select in the final part of the macro. If you need to provide Browse option to select multiple users. The last part of the macro 'True' decides whether this is single or multi valued. default is single. 
    [Macro_User_OUPicker_Name]     -> Provide option to browse user from ADUserInfo and return single 'Name'
    [Macro_User_OUPicker_Name_True]-> Provide option to browse users from AD and return 'Name' collection as Powershell Array.
  • Language Picker: This macro loads DropDown control with a options of language details. 
    [Macro_Language_LanguagePicker]     -> DropDown control loads language details and return language culture code
  • HelpText: If you need show message or tips, add below macro to load help text. 
    [Macro_Example help text here_HelpText]                          -> Load help text message 'Example help text here'.
  • External Resourse: Use below macros to get CSMO binaries 
    [Macro_Include_CSMOAssembly]        -> Include Csom Assembly's
    [Macro_Include_SPOAdminUrl]         -> Include Sharepoint Tanent Admin Url

    How to configure advanced Macro in Actions?

  • Powershell Modules: Use below macros to load required PowerShell modules. 
    [Macro_LoadPSModule_ExchangeOnline] -> Office 365 Exchange Module
    [Macro_LoadPSModule_MSOnline]       -> Azure AD Module
    [Macro_LoadPSModule_SPOnline]       -> Sharepoint Online Module
  • Credential: Use below macros to load credentials in different ways. 
    [Macro_Include_SecureCredential]    -> Include UserName, Password, Password SecureString
    [Macro_Include_PSCredential]        -> Include Powershell Credential
